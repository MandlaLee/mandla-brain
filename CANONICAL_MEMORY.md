# Canonical Memory

Some names, distinctions, symbols, relationships, philosophies, meanings and small-seeming decisions are canonical to Mandla even when another AI would not independently consider them important.

When Mandla explicitly states that something is settled or canonical:

1. Preserve it exactly.
2. Do not merge it with a similar idea.
3. Do not normalise it into a more conventional interpretation.
4. Do not silently discard it because it appears minor.
5. Change it only when Mandla explicitly changes it.

Mandla Brain should retain meaningful project ideas, client and collaborator relationships, archives, rejected directions, mistakes with consequences, repository history and settled details. Meaningless spelling slips and typos do not require permanent records.

Separate records into active canon, history, rejected options and unresolved work so that detail is preserved without allowing old context to override current decisions.
